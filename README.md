# RosATest - Приложение для запросов справок

## Контекст и задача

### Контекст
Сотрудники компании регулярно запрашивают бухгалтерские справки: 2-НДФЛ, о месте работы и стаже, о среднем заработке, а также справки произвольного типа. Сейчас процесс не автоматизирован, что создаёт задержки, дубликаты и сложности с отслеживанием информации о готовности.
Необходимо реализовать сервис, который позволит сотрудникам подавать запросы, отслеживать их статус, а бухгалтерам — обрабатывать заявки в единой очереди с фиксацией всех изменений. Ключевые ожидания к решению: предсказуемость бизнес-логики и защита от повторных отправок.

### Задача
В рамках тестового задания мы моделируем упрощённый контур этого процесса. Ваша задача — реализовать ядро сервиса, сохранив баланс между скоростью разработки, читаемостью кода и надёжностью бизнес-логики.
Бизнес-требования:
- как сотрудник, я хочу отправить запрос на получение справки, чтобы оформить справку дистанционно без визита в бухгалтерию;
- как сотруднику, я хочу посмотреть свои отправленные запросы, чтобы отслеживать их статус;
- как бухгалтер, я хочу увидеть детали запроса (сотрудник, вид справки, количество экземпляров и причина запроса), чтобы ускорить оформление справки;
- как бухгалтер, я хочу менять статус запросов, чтобы оперативно информировать сотрудника и фиксировать этап работы.
Не нужно сильно углубляться в проработку UI, будет оцениваться UX.

## Функционал

1. Подсистема идентификации, аутентификации, авторизации
2. Условное отображение возможностей для разных групп пользователей (создание запроса для сотрудника, изменение запроса для бухгалтера)
2. Создание запроса с предложенным или произвольным типом, количеством, причиной
3. Возможность изменения статуса запроса бухгалтером ("В процессе", "Готово", "Отклонена")
4. Принудительная фильтрация списка запросов для пользователя 
(для сотрудника только свои, для бухгалтера все) и опциональная по статусу
5. Обработка случая, при котором есть запрос 
с определенным типом в статусе "В процессе" - система не даст создать новый пока предыдущий не будет готов

## Установка и запуск

```bash
# Windows
dotnet publish -c Debug -r win-x64 --self-contained true \
  -p:PublishSingleFile=true \
  -p:DebugType=full \
  -o ./publish/debug/win-x64

# macOS (Intel)
dotnet publish -c Debug -r osx-x64 --self-contained true \
  -p:PublishSingleFile=true \
  -p:DebugType=full \
  -o ./publish/debug/osx-x64

# macOS (Apple Silicon)
dotnet publish -c Debug -r osx-arm64 --self-contained true \
  -p:PublishSingleFile=true \
  -p:DebugType=full \
  -o ./publish/debug/osx-arm64

# Linux
dotnet publish -c Debug -r linux-x64 --self-contained true \
  -p:PublishSingleFile=true \
  -p:DebugType=full \
  -o ./publish/debug/linux-x64
```

## Настройка

- Никакой ручной настройки
